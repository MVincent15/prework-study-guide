HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The h1/h2 ... elements represent the level of ehading a given text block represents.
  • The p element represents a paragraoh or block of text
  • The ul,ol,li elements represent unordered, ordered and list items.
  • The img element contains information about images that are displayed on the webpage. The src attrubite is important becuase it defines the lcoation of the image.
  • The br element creates a line break
  • The a element creates a link to the same webpage or other webpages. The href attribute points to the URL for the link
  • The style element defined the CSS style associated with a website
  • The meta element defiens teh metadata such as the character set, kewords, description, author and viewpoint
  • The script element embeds client side scripts in teh HTML document

CSS

  • A margin indicates how much space we want around the outside fo an element
  • A padding indicated how much space we want around the content inside an element
  • Internal style sheet contains CSS rules for the webpage in teh head section of the HTML file
  • External CSS style sheets allow developers to keep all their CSS rules in a seperate file, making changes easier
  • A selector defines the element or attributes to which the rules or declarations will apply
  • Declarations contain the CSS property we want to apply and the value of the property
  • You can combine two selectors with a single (,) allowing declarations to affect both selectors
  • When you use an external CSS, always remember to link to the HTML otherwise you will not see the changes
  • Class attributes allows us to share a CSS rule to any elelmt we choose by assigning the rule to a class attribute with a class selector.
  • The Class attribute can be assigned to the elemtn in HTML, enabling us to freely choose which elemtns will be assigned in the CSS

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new bracnh and switches to it
  • git add -A stages your changes
  • git commit -m "your message here" descibes the changes you made
  • git pull origin main- pulls in the base branch to your feature branch. Making sure its up to date
  • git push origin feature/(name of feature) pushes the changes to the reomte repo
  • Using git commands we can intiate a repo, add our code to the repo, and view different versions which are each stored in a unique commit
  • Repo is a folder in which all files and thier version histories are stored
  • A branch is a workspace in which you can make changes that won't affect the live site
  • A pull request (PR) is a way to ask for changes made to a branch to be merged into another branch.
  • A merge is after a pull request is approved the commit, it will be merged from one branch to another and tehn deployed on the live site
  • Issues are how work is tracked using Git. Allowing users to report new tasks and fixes, as well as track progress
  • Always remember to git pull to make sure that you have the latest version of the project on the local computer before you start
